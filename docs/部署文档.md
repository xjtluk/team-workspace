# BKS Team Workspace 部署文档

## 环境要求

| 项目 | 要求 |
|------|------|
| OS | Windows 10/11, Linux, macOS |
| Node.js | >= 18.0.0 |
| npm | >= 9.0.0 |
| 磁盘 | >= 500MB（含 node_modules） |
| 内存 | >= 512MB（推荐 1GB+） |
| 端口 | 3210（默认，可通过 PORT 环境变量修改） |

## 安装步骤

### 1. 克隆仓库

```bash
cd D:/BKS/projects
git clone <repo-url> team-workspace
cd team-workspace
```

### 2. 安装依赖

```bash
npm install
```

### 3. 配置环境变量

创建 `.env` 文件：

```bash
# SiliconFlow（主力模型）
SILICONFLOW_API_KEY=sk-xxx

# 火山方舟（ByteDance）
ARK_API_KEY=ark-xxx

# 智谱（CX 用）
ZHIPU_API_KEY_CX=xxx

# 智谱（小马用）
ZHIPU_API_KEY_XIAOMA=xxx

# MiMo（小米）
XIAOMI_API_KEY=sk-xxx
```

### 4. 构建前端

```bash
npm run build
```

### 5. 启动服务

```bash
# 方式一：直接启动
npm start

# 方式二：watchdog 守护（推荐）
npm run watchdog

# 方式三：开发模式
npm run dev
```

### 6. 启动 Agent

每个 Agent 在单独的终端中启动：

```bash
node start-cc.mjs       # CC Agent
node start-cx.mjs       # CX Agent
node start-xiaoma.mjs   # 小马 Agent
```

或使用 watchdog 一键启动所有服务 + Agent：

```bash
npm run watchdog
```

## 验证部署

1. 访问 http://localhost:3210 — 应显示像素风格界面
2. 访问 http://localhost:3210/api/health — 应返回 `{"status":"ok",...}`
3. 在群聊中发送消息 — Agent 应自动回复

## 配置说明

### 端口配置

默认端口 3210，修改方式：

```bash
PORT=8080 npm start
```

### 模型配置

模型选择在各启动脚本中配置：

| 启动脚本 | 默认模型 | 说明 |
|----------|---------|------|
| start-cc.mjs | Claude Code | 从 Claude 配置文件加载 |
| start-cx.mjs | DeepSeek V4 Pro | 可切换 GLM-4.7（[困难] 标记） |
| start-xiaoma.mjs | GLM-4.7-Flash | 免费，项目管理够用 |

### 数据库

SQLite 数据库位于 `data/workspace.db`，启用 WAL 模式。

备份：
```bash
npm run backup
```

恢复：
```bash
node scripts/backup-db.mjs --restore workspace_2026-06-03_120000.db
```

## 生产环境建议

### 使用 Watchdog

watchdog 替代 PM2（Windows 上 PM2 损坏），提供：
- 崩溃自动重启
- HTTP 健康探针
- 每日数据库备份
- 指数退避重启策略

```bash
npm run watchdog
```

### Windows 开机自启

创建快捷方式或使用 Windows 任务计划程序：

1. 创建 `start-all.bat`：
```bat
cd /d D:\BKS\projects\team-workspace
node scripts\watchdog.mjs
```

2. 将快捷方式放入 `shell:startup` 文件夹

### 日志管理

日志在 `logs/` 目录，建议定期清理或配置日志轮转。

### 数据库备份策略

watchdog 每日自动备份，保留 30 份。如需更频繁备份：

```bash
# 手动备份
npm run backup

# 或添加 Windows 任务计划程序定时执行
```

## 故障排除

### 服务启动失败

1. 检查端口 3210 是否被占用：`netstat -ano | findstr :3210`
2. 检查 `.env` 文件是否存在且 Key 正确
3. 检查 Node.js 版本：`node --version`

### Agent 不响应

1. 检查 Agent 进程是否在运行
2. 检查 Agent 终端是否有错误输出
3. 检查 API Key 是否有效
4. 访问 /api/health 确认服务器正常

### 数据库锁定

如果出现 SQLITE_BUSY 错误：
1. 确认 WAL 模式已启用（db.js 中配置）
2. 检查是否有多个进程同时写入
3. 重启服务

### 内存占用过高

1. 检查 `ecosystem.config.cjs` 中 `max_memory_restart` 设置
2. watchdog 默认无内存限制，可手动添加
3. 定期重启服务释放内存

## 回滚方案

1. 停止 watchdog（Ctrl+C）
2. 恢复数据库备份：`node scripts/backup-db.mjs --restore <备份文件>`
3. 如需回滚代码：`git checkout <tag/commit>`
4. 重新启动：`npm run watchdog`
