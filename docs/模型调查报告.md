## 二、各模型详评

### 2.1 DeepSeek-V4-Pro — 全能旗舰

**优势**：
- SWE-bench Pro 55.4%、LiveCodeBench 93.5，编码能力在可用模型中排第一
- Terminal-Bench 2.0 67.9，Agent 能力是可用模型中最高
- 1M 上下文 + 384K 输出，长代码库分析和重构无压力
- 支持 Think Max 模式，推理深度最高
- MIT 开源

**劣势**：
- 价格是 Flash 的 12 倍
- 49B 激活参数，速度是 Flash 的 1/4
- 在硅基和火山两边都有，主力用量建议走硅基（2000万永久免费）

**推荐场景**：CC 架构设计、技术决策、复杂代码审查、超长上下文分析

---

### 2.2 DeepSeek-V4-Flash — 性价比之王

**优势**：
- V4-Pro 的 80-90% 能力，1/12 价格
- Flash-Max 模式下编码接近 Pro（SWE-bench Pro 52.6% vs 55.4%，LiveCodeBench 91.6 vs 93.5）
- 速度是 Pro 的 3-4 倍，适合高频调用
- 1M 上下文同 Pro
- MIT 开源

**劣势**：
- 世界知识和事实准确性明显弱于 Pro（SimpleQA 34.1% vs 57.9%）
- 复杂 Agent 任务有差距（Terminal-Bench 56.9 vs 67.9）
- 长上下文检索略弱

**推荐场景**：CX 主力编码、CC 日常脚本、高频批量任务

**关键洞察**：Flash-Max（开启思考模式）能力大幅接近 Pro，80% 的活能用 Flash 代替 Pro，但 20% 的复杂 Bug 修复和架构分析必须 Pro。

---

### 2.3 DeepSeek-V3.2 — 上代旗舰

**优势**：
- 671B 参数，37B 激活，架构成熟稳定
- 128K 上下文够用
- 通用编码能力强

**劣势**：
- 上下文窗口（128K）远小于 V4（1M）
- 无 V4 的混合注意力架构
- 编码和 Agent 能力均弱于 V4 系列
- 仅在火山方舟可用，50万 tokens 额度有限

**推荐场景**：CX 编码备选（当 V4-flash 不可用时）

---

### 2.4 GLM-4.7 — 智谱旗舰，开源编码 SOTA

**优势**：
- SWE-bench Verified 73.8%，LiveCodeBench V6 84.9%，在开源模型中刷新 SOTA
- LiveCodeBench V6 超越 Claude Sonnet 4.5
- Code Arena 盲测开源第一
- 工具调用 τ²-Bench 87.4%，Agent 能力突出
- 交错式思考 + 保留式思考 + 轮级思考，创新推理机制
- 智谱 500万一次性 + 火山 50万+续，双平台可用

**劣势**：
- 智谱额度是一次性的，用完即止
- SWE-bench Pro 数据未公开，复杂软件工程能力待验证
- 总体弱于 DeepSeek-V4-Pro 旗舰

**推荐场景**：CC 复杂架构任务（高价值场景，值得消耗智谱储备）、CX 复杂编码（火山续期额度优先）

---

### 2.5 Doubao-Seed-2.0-pro — 字节旗舰，编码质量最优

**优势**：
- 2026年5月中国大模型评测：代码生成场景评分 85.7，质量评分 88.3，**编码质量在国产模型中排名第一**
- 可用率 100%，稳定可靠
- 数学推理出色（场景评分 78.7）
- 仅火山方舟可用，可续期

**劣势**：
- 非开源
- 缺乏 SWE-bench Pro 等国际权威评测数据
- 仅火山平台，初始 50万额度有限

**推荐场景**：CX 复杂编码任务（质量优先时）、CC 备选推理

---

### 2.6 Doubao-Seed-2.0-code — 代码专项

**优势**：
- 编码专项优化：推理速度更快、代码格式更规范
- 比 Pro 便宜约 35%
- LiveCodeBench v6 87.8，与 Pro 持平
- SWE-bench Pro 约 58%（推算），编码专项能力接近旗舰
- 支持 Fill-in-the-middle（FIM），代码补全场景专用

**劣势**：
- 纯文本任务不如 Pro（创意写作等）
- 仅火山方舟可用

**推荐场景**：CX 代码生成主力（编码专项 + 成本更低 + 格式更规范）、批量重构、代码补全

---

### 2.7 Doubao-Seed-2.0-lite — 全模态轻量

**优势**：
- 全模态理解（视频/图片/音频/文本），我们池子里唯一的多模态模型
- SWE-bench Verified 73.5%，LiveCodeBench v6 81.7，编码能力超预期
- ClawBench 综合排名全球第二（93.1分），性价比全球第一（282.1分）
- 1/5 Pro 的价格
- GUI 能力突出，适合端侧操作

**劣势**：
- 代码生成是附带能力，非主打
- 不如 Seed-2.0-code / Pro 编码深度强
- 复杂算法实现力有不逮

**推荐场景**：截图分析、UI审查、前端复刻、端侧操作辅助（非纯编码场景）

---

### 2.8 MiMo 2.5 Pro — 小米旗舰，全模态兜底

**优势**：
- 1.02T 参数，42B 激活，1M 上下文，旗舰级配置
- SWE-Bench Pro 57.2，Coding Agent 能力与 DeepSeek-V4-Pro 同档
- MiMo Coding Bench 68.4，综合编码能力强
- Terminal-Bench 2.0 #3.4，Agent 能力优秀
- 全模态支持
- **余额充足，随便用**，是最可靠的兜底

**劣势**：
- 编码峰值不如 DeepSeek-V4-Pro-Max
- 小米 API 稳定性需持续观察

**推荐场景**：CC 日常主力备用、最终兜底、多模态截图审查、其他平台全挂时的保底方案

---

### 2.9 GLM-4.5-Air — 轻量 Agent 特化

**优势**：
- 106B 总参 / 12B 激活，极致轻量
- 工具调用成功率 90.6%，Agent 场景表现接近旗舰
- 价格极低（输入 ¥0.14/M）
- 智谱 1200万一次性额度
- 128K 上下文

**劣势**：
- SWE-bench Verified 仅 57.6%，复杂编码能力弱
- 一次性额度不可续
- 不适合深度代码任务

**推荐场景**：CC 日常简单任务、文档处理、轻量 Agent 调度（非核心编码）

## 五、最终建议

### 5.1 推荐策略：三层模型架构

```
第一层（主力）：硅基 DeepSeek V4-Pro + V4-Flash（2000万免费，覆盖CC+CX 80%场景）
第二层（专项）：火山 Seed-2.0-code（CX编码专项）+ Seed-2.0-lite（多模态）+ GLM-4.7（高价值推理）
第三层（兜底）：MiMo 2.5 Pro（拉满，哪里挂补哪里）
```

### 5.2 CC 推荐模型优先级

```
日常: V4-Pro(硅基) → V4-Flash(硅基) → MiMo 2.5 Pro
复杂: V4-Pro-Max(硅基) → MiMo 2.5 Pro → GLM-4.7(火山优先)
多模态: Seed-2.0-lite(火山) → MiMo 2.5 Pro
兜底: MiMo 2.5 Pro → Anthropic原生
```

### 5.3 CX 推荐模型优先级

```
日常编码: V4-Flash(硅基) → Seed-2.0-code(火山) → V4-Pro(硅基)
复杂编码: V4-Pro(硅基) → Seed-2.0-code(火山) → MiMo 2.5 Pro
批量任务: V4-Flash-Max(硅基) → V3.2(火山) → GLM-4.5-Air(智谱)
兜底: V4-Flash(火山) → MiMo 2.5 Pro
```

### 5.4 核心原则

1. **DeepSeek-V4-Flash 扛日常**：80% 的任务它都能搞定，成本极低
2. **DeepSeek-V4-Pro 攻难点**：复杂架构、深度审查、Bug 修复才上
3. **Seed-2.0-code 做专项**：CX 编码主力备选，代码格式更规范、成本更低
4. **MiMo 2.5 Pro 当保险**：余额充足随便用，任何平台挂掉立刻顶上
5. **智谱额度当黄金**：GLM-4.7 只用于高价值任务，坚决省着花
6. **火山额度按需续**：50万起步不够找 KK 续 200万/天，不用省
7. **多平台重叠模型走主力平台**：DeepSeek 走硅基、GLM-4.7 走火山
