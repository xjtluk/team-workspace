## 二、CC 侧意见

### 2.1 降级链简化建议

**当前问题**：降级链条过长（5-6级），中间环节实际价值不大。

**建议CC降级链**：
```
日常: V4-Pro(硅基) → V4-Flash(硅基) → MiMo 2.5 Pro
复杂: V4-Pro(硅基) → MiMo 2.5 Pro → GLM-4.7(火山)
多模态: Seed-2.0-lite(火山) → MiMo 2.5 Pro
兜底: MiMo 2.5 Pro → Anthropic原生
```

**理由**：
- 火山V4-Pro和硅基V4-Pro是同一模型，硅基免费额度更充裕，没必要走火山中转
- Doubao-Pro在CC场景下没有明确优势（CC需要的是推理深度，不是编码速度）
- MiMo的SWE-bench Pro 57.2与V4-Pro同档，作为第二选择比中间过渡模型更合理

### 2.2 V4-Pro-Max 问题

报告提到V4-Pro-Max（SWE-bench 80.6%），但当前配置中没有。需要确认：
- 这是独立的endpoint/model ID，还是V4-Pro开启Max思考模式？
- 如果是模式切换，需要在调用时加参数，不是模型选择问题
- **暂不处理**，等确认后再加入配置

### 2.3 MiMo 使用策略

报告建议MiMo更积极使用（CC日常备用），我同意：
- MiMo余额充足，不用担心成本
- SWE-bench Pro 57.2，编码能力够用
- 当硅基不可用时，直接切MiMo比切火山更简单

### 2.4 智谱额度管理

报告建议GLM-4.7只用于高价值任务，我完全同意：
- 智谱是一次性额度，用完即止
- 火山侧GLM-4.7可续额度，优先走火山
- 智谱47和46只在火山+硅基同时不可用时才动用
