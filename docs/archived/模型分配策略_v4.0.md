# 模型分配策略 v4.0

**制定人**: CC
**依据**: 小马调研报告 + model-allocation.yaml v3.0 + 实际使用反馈
**日期**: 2026-06-03
**状态**: CC侧完成，CX侧未反馈（会话被取消），待CX确认后生效

---

## 核心变化（v3.0 → v4.0）

| 变化 | v3.0 | v4.0 | 原因 |
|------|------|------|------|
| CC降级链 | 6级（含火山Doubao中转） | 4级（直接MiMo） | 中间过渡层无实际价值 |
| CX主力 | V4-Flash为主 | V4-Flash + Seed-2.0-code双主力 | Seed-2.0-code编码专项更强 |
| MiMo定位 | 最终兜底 | 第二/第三选择 | 余额充足+能力够用，应积极使用 |
| 智谱 | 在常规降级链中 | 退出常规链，纯战略储备 | 一次性额度不可续 |
| 火山Doubao | CC/CX均有 | CX编码保留，CC移除 | CC需要推理深度不是编码速度 |

---

## CC 降级链

```
日常/脚本:  V4-Flash(硅基) → MiMo 2.5 Pro
架构/审查:  V4-Pro(硅基) → MiMo 2.5 Pro → 火山V4-Pro
多模态:     Seed-2.0-lite(火山) → MiMo 2.5 Pro
最终兜底:   MiMo 2.5 Pro → Anthropic原生
```

**砍掉的**：火山Doubao-Pro（CC场景无明确优势）、火山V3.2（上代旗舰无必要）

---

## CX 降级链

```
日常编码:   V4-Flash(硅基) → Seed-2.0-code(火山) → 火山V4-Flash
复杂编码:   V4-Pro(硅基) → Seed-2.0-code(火山) → MiMo 2.5 Pro
批量重构:   V4-Flash(硅基) → Seed-2.0-code(火山)
代码补全:   Seed-2.0-code(火山，FIM专项)
最终兜底:   火山V4-Flash → MiMo 2.5 Pro
```

**关键变化**：Seed-2.0-code从第三级提前到第二级，与V4-Flash形成双主力

## 额度策略

| 平台 | 用途 | 消耗策略 |
|------|------|---------|
| 硅基(2000万永久) | CC+CX DeepSeek主力 | 日常消耗，无需省 |
| 火山(50万+200万/天) | Seed-2.0-code + GLM-4.7 | 按需续期，找KK |
| MiMo(余额充足) | CC备用+兜底 | 随便用，不心疼 |
| 智谱(一次性) | 纯战略储备 | 只在火山+硅基同时不可用时动用 |

**结论**：CC与小马报告建议基本一致，无重大分歧。
