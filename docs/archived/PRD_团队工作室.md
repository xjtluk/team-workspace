# PRD：Team Workspace 团队工作室

> **文档版本：** v1.1
> **编写人：** 小马（项目部 Leader）
> **日期：** 2026-05-31
> **评审人：** CC（研发部 Leader）
> **状态：** 已更新（三方→N方、P0提升、扩展性约束）

---

## 1. 产品背景

### 1.1 问题陈述

Marvis 团队（老板 KK、项目部小马、研发部 CC）日常协作依赖对话窗口，存在以下痛点：

1. **状态不可见**：不知道对方当前在忙什么、进度如何，只能靠口头问
2. **信息分散**：PRD、技术方案、测试报告等产出物散落在不同轮对话中，缺乏统一的文档索引
3. **缺乏仪式感**：虽然是三人小团队，但没有一个"在一起工作"的空间感

### 1.2 用户画像

| 角色 | 身份 | 核心需求 |
|------|------|------|
| 老板（KK） | 决策者 | 一眼看到团队成员在做什么，随时参与讨论，拍板决策 |
| 小马 | 项目部 Leader | 汇报进度、调度资源、参与讨论、产出文档 |
| CC | 研发部 Leader | 可视化工作状态、接收任务、产出代码和文档 |

---

## 2. 产品目标

### 2.1 核心目标

构建一个轻量级本地团队协作平台——**Team Workspace（团队工作室）**，以像素风 2D 界面聚合团队工作状态和即时沟通。

### 2.2 成功指标

| 指标 | 目标值 |
|------|------|
| 成员状态可见性 | 3 人均能在面板上看到彼此实时状态和进度气泡 |
| 沟通即时性 | 群聊消息延迟 < 1 秒 |
| 部署复杂度 | 单命令启动，无需外部依赖和服务 |
| MVP 交付周期 | 8 天内（4 阶段） |
| 产出物完整性 | 所有项目文档落地为本地文件，可按文件名检索 |

---

## 3. MVP 功能范围

### 3.1 包含功能

| 模块 | 功能描述 | 优先级 |
|------|------|:--:|
| **像素办公室面板** | 2D 像素风办公场景，含工作区/休息区/调试区 | P0 |
| **角色状态展示** | CC（Clawd 像素螃蟹）、小马（像素小马），5 种状态+气泡文字+进度条 | P0 |
| **角色移动动画** | 根据状态自动移动到对应区域，行走过渡动画 | P1 |
| **团队群聊** | N方群聊（当前成员：KK/CC/小马），消息发送/接收/历史记录，新成员无缝加入 | P0 |
| **Agent 接入** | CC 和小马通过 HTTP API 上报状态和发送消息 | P2 |
| **离线处理** | 心跳保活 + 离线标记 + 消息排队 + 上线拉取 | P2 |

### 3.2 非 MVP 范围

- 权限认证与用户管理
- 文件共享 / 富媒体消息
- 多人视频/语音
- 移动端适配
- 云端部署
- 迷你地图

### 3.3 新成员扩展性约束

团队工作室设计从 Day 1 即支持未来新成员加入。核心约束：

1. **Agent 注册机制数据驱动**：Agent 表结构和注册逻辑不硬编码特定成员。新增成员仅需（1）在 agents 表插入一行配置记录；（2）提供对应的像素网格数据文件；（3）前端自动识别并渲染新角色
2. **N 方群聊**：聊天面板支持任意数量成员同时在线，不设上限
3. **零代码变更**：新增成员不需要修改前端或后端代码，完全通过配置和数据文件驱动

## 5. 视觉风格

### 5.1 整体风格

- **像素风（Pixel Art）**：8-bit 复古游戏风格
- **暗色主题**：深色背景 + CRT 扫描线效果
- **字体**：Press Start 2P（像素字体）

### 5.2 角色设计（已通过原型验证）

| 角色 | 形象 | 渲染方式 | 备注 |
|------|------|------|------|
| CC | Clawd 像素螃蟹 | 左侧办公室：CSS 像素方块手绘；右侧聊天：Clawd 图片头像 | Claude Code 官方吉祥物 |
| 小马 | Marvis 像素小马 | 左侧办公室：CSS 像素方块手绘；右侧聊天：🐴 emoji | 橙色身体 + 棕色鬃毛 |
| KK（老板） | 👑 | 聊天靠右对齐，金色标识 | 决策者 |

### 5.3 布局

```

│   ├ 休息区 (sofa)    │   ├ KK 发言（靠右金色）    │
│   ├ 工作区 (desk)    │   ├ CC/小马发言（靠左）    │
```

### 5.4 视觉升级约束

像素面板和角色动画是团队工作室的核心体验载体，不可因功能开发进度压力而简化或降级：

1. **像素面板持续升级**：CSS 像素方块手绘风格保持完整，后续迭代中持续优化细节（光影、粒子效果等）
2. **角色动画不砍**：状态切换的行走过渡动画、工作/说话/错误动画不得因赶工期而省略或降级为静态切换
3. **原型保真**：交互原型中的所有视觉效果（CRT 扫描线、像素字体、暗色主题）必须在 MVP 中完整保留

## 6. 角色状态机

| 状态 | 触发条件 | 面板位置 | 动画 | 气泡 |
|------|------|------|------|------|
| **待机 (idle)** | 无任务 | 休息区 | 站立 | 无 |
| **工作中 (working)** | 任务开始 | 工作区 | 工作 | 活动描述 + 进度条 |
| **讨论中 (talking)** | 参与群聊讨论 | 工作区 | 说话 | 讨论内容 |
| **出错 (error)** | 任务异常 | 调试区 | 错误闪烁 | 错误描述 |
| **离线 (offline)** | 心跳超时 60s | 休息区 | 离线/灰度 | "离线中" |

状态切换时角色自动行走移动到对应区域。
