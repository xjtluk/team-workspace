# BKS Team Workspace — Portfolio

> 团队协作平台项目展示。版本：V1.1

---

## 项目概述

BKS Team Workspace 是一个面向 AI Agent 团队的像素风格实时协作平台。支持多个 AI Agent（CC、CX、小马）在同一像素办公室中协作完成软件开发任务。

## 技术架构

```
┌─ 浏览器 (localhost:3210) ─────────────────────────────┐
│  ┌─ PixelOffice (Canvas 1000x700) ──────────────────┐ │
│  │  工作区 │ 休息区 │ 调试区 │ 角色动画 │ 状态气泡 │   │
│  └──────────────────────────────────────────────────┘ │
│  ┌─ ChatPanel (WebSocket 实时) ──────────────────────┐ │
│  │  N方群聊 │ 消息历史 │ 离线排队 │ 在线状态标记    │ │
│  └──────────────────────────────────────────────────┘ │
│  ┌─ Dashboard (状态看板) ───────────────────────────┐ │
│  │  中文状态 │ 活动概况 │ 进度条 │ InfoPanel 悬浮   │ │
│  └──────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
                        ↕ WebSocket
┌─ Express 后端 ──────────────────────────────────────────┐
│  REST API │ WebSocket Hub │ SQLite (WAL) │ Watchdog    │
│  /api/health │ /api/heartbeat │ /api/messages          │
│  /api/agents │ Provider Cooling │ Adaptive Timeout     │
└─────────────────────────────────────────────────────────┘
                        ↕ HTTP
┌─ AI Agent Layer ────────────────────────────────────────┐
│  CC (MiMo 2.5 Pro)     CX (DeepSeek V4 Pro)            │
│  小马 AI (GLM-4.7-Flash)                               │
│  Agent SDK: ai-reply.js │ agent-client.js │ memory.js  │
└─────────────────────────────────────────────────────────┘
```

## 功能列表

### 像素办公室面板
- 2D 像素风办公室场景（1000x700，6 工位 3x2 布局）
- 3 个区域：工作区 / 休息区 / 调试区
- 角色精灵：CC（Clawd 像素螃蟹）、CX、小马（像素小马）
- 5 种工作状态：工作中 / 空闲中 / 讨论中 / 异常 / 离线
- 动画层：服务器灯、时钟、咖啡、饮水机、街机

### 团队群聊
- N 方实时群聊（WebSocket）
- 消息发送 / 接收 / 历史记录（SQLite 持久化）
- 离线消息排队 + 上线拉取
- 在线状态实时标记

### 状态看板
- 看板卡片：名称 / 角色 / 状态（中文）/ 进度条
- InfoPanel 悬浮信息面板
- Activity 中文概况（12 字截断）
- 离线超过 60s 显示"重启中"

### AI Agent 基础设施
- Agent SDK（ai-reply.js）：工具调用 / 安全校验 / 超时控制
- 多模型降级链：SiliconFlow → 火山方舟 → 智谱 → MiMo
- Provider 冷却机制（429 后 5 分钟冷却）
- 自适应超时（工具调用 30s + 整体 120s 兜底）
- Agent 心跳追踪 + 健康检查
- Watchdog 进程守护 + 自动重启

### 协作流程
- Hub-Spoke 架构：小马（Hub）→ CC / CX（Spokes）
- CC-CX 分工铁律（设计/实现分离）
- CC-CX 互助铁律（技术问题内部消化）
- TOK 自检机制（Task / Operation / Key Result）
- Karpathy 编码四原则（思考先于编码 / 简洁优先 / 手术式修改 / 目标驱动）
