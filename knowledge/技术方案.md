## 一、技术栈

| 层级 | 选型 | 版本 | 用途 |
|------|------|------|------|
| 后端运行时 | Node.js | 18+ | 事件驱动，适合 WebSocket 长连接 |
| HTTP 框架 | Express | 4.x | 轻量 HTTP 服务 + 静态文件 |
| WebSocket | ws | 8.x | 实时通信，无 socket.io 额外负担 |
| 数据库 | SQLite (sql.js) | 1.x | 零配置单文件，同步 API |
| 前端框架 | Preact | 10.x | 3KB gzip，API 兼容 React |
| 构建工具 | Vite | 6.x | 极速 HMR，ESM 原生 |
| Canvas 渲染 | 原生 Canvas 2D | — | 像素背景 + 角色逐格渲染 |
| Markdown | marked + DOMPurify | — | 消息 Markdown 解析 + XSS 过滤 |

## 三、核心 API

### 3.1 Agent 状态上报

```
POST /api/status
Body: { agentId, status, activity?, progress?, location? }
Status: idle | working | talking | error | offline
Location: desk | sofa | bug
```

### 3.2 Agent 消息发送

```
POST /api/message
Body: { from, content, type?, replyTo? }
Type: text | status_update
```

### 3.3 心跳保活

```
POST /api/heartbeat
Body: { agentId }
Response: { ok, pendingMessages, serverTimestamp }
```

### 3.4 聊天历史查询

```
GET /api/history?limit=50&before=msg_xxxxx
Response: { messages: [...], hasMore: bool }
```

### 3.5 Agent 注册（新成员）

```
POST /api/register
Body: { id, name, agentType, color?, gridFile? }
Response: { ok, agent: {...} }
```

### 3.6 健康检查

```
GET /api/health
Response: { status, uptime, agents: { cc: { online, healthy, lastSeen, ... }, ... } }
```

## 四、WebSocket 事件协议

### Server → Client

| 事件类型 | 说明 |
|---------|------|
| `status_change` | Agent 状态变更广播 |
| `new_message` | 新消息推送 |
| `agent_online` / `agent_offline` | 成员上下线 |
| `agent_registered` | 新成员注册通知 |
| `message_ack` | 消息送达确认 |
| `ping` | 心跳探测 |

### Client → Server

| 事件类型 | 说明 |
|---------|------|
| `send_message` | 发送聊天消息 |
| `pong` | 心跳响应 |

## 五、数据库设计

### agents 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | 唯一标识（cc/xiaoma/kk/cx） |
| name | TEXT | 显示名称 |
| agent_type | TEXT | agent(有状态上报) / human(仅聊天) |
| color | TEXT | HEX 角色主色 |
| grid_file | TEXT | 像素网格文件路径（human=NULL） |
| current_status | TEXT | idle/working/talking/error/offline |
| current_activity | TEXT | 气泡文字 |
| progress | INTEGER | 进度 0-100 |
| location | TEXT | desk/sofa/bug |
| online | INTEGER | 0/1 |
| last_seen | INTEGER | 最后心跳时间戳(ms) |

### messages 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | msg_{timestamp}_{agentId} |
| from_id | TEXT FK | 发送者 |
| from_name | TEXT | 发送者显示名 |
| content | TEXT | 消息内容（支持 Markdown） |
| type | TEXT | text/system/status_update |
| reply_to | TEXT | 回复消息 ID |
| created_at | INTEGER | 时间戳 |

### status_log / offline_queue 表

- **status_log**: agent_id, status, activity, progress, location, created_at（审计追踪）
- **offline_queue**: to_id, message_id, delivered(0/1), created_at（离线消息排队）

## 六、角色像素网格渲染

### 渲染方式
- 每个角色由二维数组定义像素网格（如 Clawd 螃蟹 12×10 格）
- 每格 6×6px，Canvas 2D 逐格渲染
- `imageSmoothingEnabled=false` 保证像素精准对齐

### 配色映射
| 索引 | 色值 | 用途 |
|------|------|------|
| 0 | null | 透明 |
| 1 | #E88D2A | 主体橙棕 |
| 2 | #8B4513 | 深棕（轮廓） |
| 3 | #1A1A2E | 深色瞳 |
| 4 | #A0522D | 鬃毛（小马） |
| 5 | #F5DEB3 | 浅色高光 |

### CSS 状态动画

| 状态 | Class | 动画 |
|------|-------|------|
| idle | char-idle | work-bounce 2s infinite |
| working | char-working | work-bounce 0.8s infinite |
| talking | char-talking | talk-wiggle 0.3s infinite |
| walking | char-walking | walk-bob 0.4s infinite |
| error | char-error | error-shake 0.15s infinite + brightness(0.6) |
| offline | char-offline | grayscale(100%) opacity(0.5) |

## 七、消息可靠性保障

1. **持久化优先** — 消息先写 SQLite，再 WebSocket 广播
2. **ACK 确认** — Server 收到消息后回 message_ack，客户端超时重发
3. **断线重连** — 指数退避（1s→30s），重连后拉取缺失消息
4. **离线排队** — 离线成员消息写入 offline_queue，上线后拉取
5. **送达状态** — 每条消息记录 delivered_to 成员列表

## 八、进程模型

```
单进程 (Node.js)
├── HTTP Server (Express, :3210)
├── WebSocket Server (ws)
└── SQLite Database (data/workspace.db)

Watchdog 守护进程 (scripts/watchdog.mjs)
├── 启动所有服务 (server + 3 Agent)
├── 崩溃自动重启 (指数退避 10s→300s)
├── 健康检查 (/api/health 每30s)
├── Agent 心跳检查 (每60s，连续2次异常重启)
└── 每日数据库备份 (scripts/backup-db.mjs)
```
