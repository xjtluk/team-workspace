## 一、网络代理规则

### 1.1 基本原则

- 国内站点（百度、阿里云、国内API等）→ **直连**，不走代理
- 国外站点（GitHub、Google、npm等）→ **走代理** `127.0.0.1:7897`
- Clash Verge 运行于规则模式，自动判断，无需手动区分

### 1.2 Agent 进程代理（已配置）

`ai-reply.js` 的 `UTF8_ENV` 已设置环境变量，bash 工具调用自动经过 Clash：

```
http_proxy=http://127.0.0.1:7897
https_proxy=http://127.0.0.1:7897
```

### 1.3 手动命令代理

**境外站点（加 --proxy）：**
```bash
# GitHub
curl -s --proxy http://127.0.0.1:7897 https://api.github.com/xxx

# npm
curl -s --proxy http://127.0.0.1:7897 https://registry.npmjs.org/xxx

# Google
curl -s --proxy http://127.0.0.1:7897 https://www.google.com

# Git 操作（临时设置，用完清理）
git config http.proxy http://127.0.0.1:7897
git push origin master
git config --unset http.proxy
```

**国内站点（直连，不加 --proxy）：**
```bash
curl -s https://api.siliconflow.cn/v1/xxx
curl -s https://open.bigmodel.cn/api/paas/v4/xxx
curl -s https://taotoken.net/api/v1/xxx
```

### 1.4 禁止事项

| 禁止操作 | 原因 |
|----------|------|
| 修改注册表 ProxyEnable/ProxyServer | 破坏系统代理设置，导致断网 |
| 修改 WinHTTP 代理（netsh winhttp） | 影响系统级网络 |
| 设置系统级 HTTP_PROXY 环境变量 | 影响所有程序 |
| `git config --global http.proxy` | 永久生效，忘记清理会出问题 |

## 二、模型配置

### 2.1 模型分配总览

| Agent | 角色 | 日常模型 | 代码任务模型 | 提供商 |
|-------|------|----------|-------------|--------|
| **CC** | 研发部 Leader | MiMo 2.5 Pro | MiMo 2.5 Pro | 小米 |
| **CX** | 代码工程师 | DeepSeek V4 Flash | DeepSeek V4 Pro | TaoToken / 硅基流动 |
| **小马AI** | 项目部 Leader | GLM-4.7-Flash | GLM-4.7-Flash | 智谱（免费） |

### 2.2 CX 模型配置（cx-listener.mjs）

```javascript
// 模型分层
MODEL_TIERS = {
  normal: 'glm-4.7-flash',                    // 保底模型
  taotokenFlash: 'deepseek-v4-flash',          // 日常主力
  code: 'deepseek-ai/DeepSeek-V4-Pro'         // 代码任务
};

// 选择逻辑
if (isCodeTask)  → MODEL_TIERS.code           // DeepSeek V4 Pro（硅基）
if (isNormal)    → MODEL_TIERS.taotokenFlash   // DeepSeek V4 Flash（TaoToken）
if (fallback)    → MODEL_TIERS.normal          // GLM-4.7-Flash（智谱）
```

### 2.3 小马AI 模型配置（start-xiaoma.mjs）

```javascript
OPENAI_MODEL = 'glm-4.7-flash';
OPENAI_BASE_URL = 'https://open.bigmodel.cn/api/paas/v4';
```

### 2.4 CC 模型配置（start-cc.mjs）

```javascript
// claude-code-router 配置
default: xiaomi,mimo-v2.5-pro
background: siliconflow,deepseek-ai/DeepSeek-V4-Pro
providers: siliconflow, volc_ark, zhipu, xiaomi
```

### 2.5 Provider 冷却机制

- 429（限流）错误后进入 5 分钟冷却
- 冷却期间自动切换下一个 Provider
- 请求超时时间：30秒（可配置）

## 四、项目管理工具

### 4.1 进程管理

| 工具 | 用途 | 状态 |
|------|------|------|
| PM2 | 进程守护 | ❌ Windows上损坏 |
| watchdog.mjs | PM2替代（自研） | ✅ 正常使用 |
| PID文件 | 单实例保护 | ✅ `.cc-listener.pid` / `.cx-listener.pid` |

### 4.2 Git 工作流

- 分支策略：直接 push master（单人开发阶段）
- 提交格式：`[模块] 变更描述`
- Tag 格式：`V1.0`, `V1.1` 等

### 4.3 数据库

- 使用 SQLite（sql.js WASM），WAL 模式
- 自动备份：watchdog 每日备份，保留最近30份
- 备份路径：`data/backups/`
